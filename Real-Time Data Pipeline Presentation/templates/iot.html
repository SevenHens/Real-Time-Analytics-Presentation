EC7 Overview

The goal of the EC7 project is to design data infrastructure to support the minigrid such that the different components and their communication protocols are able to be centralized into a service that can provide actionable insight and monitor preformance. The challenge is that it must support real-time analytics at lower cost and lower bandwidth than traditional solutions.

Challenges

PRIVACY/SECURITY- By existing on a decentralized infrastructure and having data on many nodes and devices, it is susceptible to attack and leaks.

SCALABILITY- The heterogeneity of the devices used, each with different preformance and energy constraints, can slow down and weaken the overall infarstrucutre compared to centralized cloud data centers

RELIABILITY- In the event that a node fails, the entire system should still maintain network topology and users should still be able to access services without interruptions

Serverless Computing

The paradaigm where functions are deployed to the cloud with zero administration to the software developer. Currently offered by traditional big data providers in addition to open-source solutions, by providing the code itself, the developer does not need to run their own servers or infrastructure.

Many solutions must provide support in the form of FDKs (kits that can functionify existing apps), CI/CD, logging/metrics and language/integration support, however the extremely low cost provides a solution to many of the challenges faced by EC7.

Non-Instrusive Load Monitoring

Techniques that attempt to analyze changes in voltage and current and match those with appliances wiether through traditional clustering or neural networks.

The idea is that by preforming the computations on low-cost sensors near the equipment, the demand response system would be both real-time and low-cost.

Traditional Big Data Solutions

Note that these solutions, while promising, are still in their infancy and are still rather unreliable. Traditional open-source data solutions such as Kafka and Spark have a history of success and ease-of-use.

As honorable mentions, technologies such as ElasticSearch(for efficient text-based querying), Hadoop (for batch processing of aggregate data), MongoDB(as a data store), and TensorFlow(for predictive anlytics) also show promise when coupled properly.
